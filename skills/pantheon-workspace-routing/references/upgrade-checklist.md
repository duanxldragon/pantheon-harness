# Upgrade Checklist

1. Identify the target `pantheon-base` tag or commit.
2. Review the changed contracts, designs, and acceptance docs in `pantheon-base`.
3. Update the business repository `docs/PROJECT_INHERITANCE.md` version reference.
4. Adjust only the affected business modules or local docs.
5. Re-run the validation set that matches the touched surface.
6. Record the upgrade note and remaining gaps.
