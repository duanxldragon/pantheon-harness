# Inheritance Harness Protocol

This repo-local note defines how a consumer repository should inherit Pantheon Harness materials.

## Source Order

1. Read the canonical method source in [patterns/README.md](../../patterns/README.md).
2. Read the workflow routing contract in [architecture/methodology/workflow-routing.md](../../architecture/methodology/workflow-routing.md).
3. Project only the required landing files into the consumer repository.

## Rule

Consumer repositories may adapt paths and tooling, but they should not fork the method semantics without recording the drift in a task packet and review artifact.
