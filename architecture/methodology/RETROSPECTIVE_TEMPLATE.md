# 月度复盘模板

> 每月抽 20 分钟填一次。不需要长篇大论，每个问题 2-3 句话即可。

## 复盘日期：YYYY-MM-DD

## 1. 本月概况

- 完成了多少任务？
- 主要涉及哪些模块层级？
- 有多少任务是 Codex 独立完成的？有多少需要反复返工？

## 2. 流程效率

| 维度 | 评价（1-5） | 说明 |
|---|---|---|
| 需求→方案的转化效率 | | |
| Claude 派任务给 Codex 的信息完整度 | | |
| Codex 产出的首次可用率 | | |
| CI/CD 反馈速度 | | |
| 验收耗时 | | |

## 3. 质量表现

- SonarQube 整体评级变化（变好/变差/持平）？
- CI 失败次数？最常见的失败原因？
- 有没有出现过生产环境问题（如果有）？

## 4. 方法论漏洞

- 有没有遇到方法论没有覆盖的情况？
- 有没有发现文档之间的描述矛盾？
- 有没有应该自动化但还在手动操作的环节？

## 样例（参考）

### 复盘日期：2026-07-15

### 1. 本月概况
完成了 12 个任务，主要涉及 system/iam 和 business/cmdb。其中 8 个 Codex 一次通过，2 个需要返工（SonarQube 假阳性处理不当），2 个还在进行中。

### 2. 流程效率
| 维度 | 评价 | 说明 |
|---|---|---|
| 需求→方案的转化效率 | 4 | 用 Task Packet 模板后 Claude 派任务快多了 |
| Codex 产出的首次可用率 | 3 | SonarQube 修复时偶尔把对的代码改坏 |
| 验收耗时 | 4 | 对着验收清单逐项查，比以前快 |

### 3. 质量表现
- SonarQube: 持平，无新增问题
- CI 失败 3 次：2 次 smoke timeout，1 次 lint
- 无生产环境问题

### 4. 方法论漏洞
- `$sonar-remediate` Skill 还没建，需要 Claude 每次手动判断假阳性
- 验收清单缺"权限回退"检查项

### 5. 本月学到的一件事
当 SonarQube 规则标记的是语言内置特性（如 Go 1.22+ 的 `for range` 变量捕获）时，不要犹豫直接排除。

### 6. 下月改进计划
| 改进项 | 优先级 |
|---|---|
| 封装 `$sonar-remediate` Skill | P0 |
| 验收清单增加"权限回退"检查 | P1 |
| 本地 SonarQube 预检脚本 | P2 |
